# 반자동(권장) 워크플로우 – 매일 1포스트 (캐러셀 4장)

**처음이면 먼저 보기:** [단계별 따라하기](sns-단계별-따라하기.md) – 계정 만들기부터 배치 실행·게시까지 순서대로 정리되어 있습니다.

캡션·이미지 프롬프트는 스크립트로 자동 생성하고, 이미지 생성 + 업로드/예약은 직접 진행하는 방식입니다.

## 2. 매일(또는 주간) 반복 루틴

### A. 콘텐츠 생성

**오늘만 할 때**
```bash
node scripts/generate-sns-daily.mjs
```

**일주일치 한 번에 생성 (예: 오늘부터 7일)**
```bash
node scripts/generate-sns-daily.mjs --days=7
```
- **더블클릭으로 실행:** `scripts/generate-sns-week.bat` 파일을 더블클릭하면 위 명령이 실행되고, **결과가 자동으로 저장**됩니다.

**저장 위치 (날짜별 폴더, 수아/이든 구분)**
- **기준 폴더:** `scripts/out/`
- **날짜별 폴더:** `scripts/out/YYYY-MM-DD/` (예: `2026-03-14`)
- **각 날짜 폴더 안에:**
  - **`sua.txt`** – 수아 게시글(캡션) + 이미지 프롬프트 4개
  - **`ethan.txt`** – 이든 게시글(캡션) + 이미지 프롬프트 4개
  - **실제 이미지 (OPENAI_API_KEY 설정 시):** `sua-1.png` ~ `sua-4.png`, `ethan-1.png` ~ `ethan-4.png` (DALL·E 3로 자동 생성 후 같은 폴더에 저장)

**이미지 자동 생성**
- **환경 변수:** `OPENAI_API_KEY` (OpenAI API 키)를 설정하면 스크립트가 DALL·E 3로 수아 4장·이든 4장을 생성해 해당 날짜 폴더에 저장합니다.
- 미설정 시에는 txt만 저장되고, 이미지는 생성하지 않습니다.

### B. 이미지 4장 생성 (수아·이든 각각)

1. 스크립트 출력에서 **수아 [이미지 4장]** 1~4번 프롬프트 복사
2. AI 이미지 도구에 1번 → 2번 → 3번 → 4번 순서로 생성 후 저장 (파일명 예: `sua-2026-03-14-1.jpg` ~ `4.jpg`)
3. 같은 방식으로 **이든 [Image 1–4]** 4장 생성 후 저장

### C. 캡션 복사

- 스크립트 출력의 **[캡션 · 한 포스트에 그대로 사용]** (수아), **[Caption · use as single post]** (이든)을 각각 복사

### D. 게시 (직접 올리기 또는 예약)

**방법 1 – 당일 직접 게시**
- 인스타그램 앱에서 **새 게시물** → **여러 장 선택** → 수아 4장 선택 → 캐러셀 순서 확인 → 캡션 붙여넣기 → 게시
- 이든 계정도 동일하게 4장 + 캡션 게시
- (인스타–페이스북 연동 시 페이스북에도 자동 공유됨)

**방법 2 – 예약 발행**
- Meta Business Suite 또는 Later/Buffer 로그인
- **인스타그램** → **예약된 게시물** → 새 게시물
- 이미지 4장 업로드(순서: 1→2→3→4), 캡션 붙여넣기, 날짜·시간 선택 후 예약
- 수아·이든 각 계정에 대해 반복

## 3. 주간 배치 예시 (일주일 치 미리 준비)

1. **월요일:** `node scripts/generate-sns-daily.mjs --days=7 > scripts/out/week.txt`
2. `week.txt`에서 Day 1~7 구간별로 수아/이든 **캡션 + 프롬프트 4개** 잘라서 메모/스프레드시트에 정리
3. AI로 Day 1 수아 4장, Day 1 이든 4장 생성 → Day 2 … 순으로 진행 (또는 날짜별 폴더에 저장)
4. 예약 도구에서 Day 1~7 각각 업로드 + 캡션 + 발행 시간 설정
